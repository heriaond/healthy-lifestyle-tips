1. Úvod

Tento dokument je projektovou dokumentací k webové aplikaci Healthy Lifestyle Tips, vytvořené v rámci předmětu Internetové technologie – klient side. Projekt byl realizován týmem tří studentů (Skupina č. 10) v průběhu akademického roku 2025/26.

Aplikace slouží jako platforma pro sdílení a objevování tipů pro zdravý životní styl. Uživatelé mohou procházet tipy rozdělené do čtyř tematických kategorií: Sleep (spánek), Nutrition (výživa), Movement (pohyb) a Stress (stres). Po přihlášení mají možnost ukládat oblíbené tipy, přidávat vlastní obsah a spravovat svůj profil.

Projekt je postaven na technologii Next.js 15 s využitím App Routeru, TypeScriptu, PostgreSQL databáze a je nasazen na cloudové platformě Vercel. Zdrojový kód je dostupný v repozitáři na platformě GitHub.

2. Cíle projektu

Hlavním cílem projektu bylo vytvořit plně funkční full-stack webovou aplikaci, která prokáže schopnost týmu pracovat s moderními webovými technologiemi. Konkrétní cíle projektu byly:

Funkční cíle

  • Umožnit uživatelům procházet tipy pro zdravý životní styl rozdělené do kategorií
  • Implementovat full-text vyhledávání a filtrování tipů
  • Zajistit bezpečnou autentizaci pomocí Google OAuth a e-mailového OTP
  • Umožnit přihlášeným uživatelům ukládat, přidávat a spravovat tipy
  • Vytvořit administrátorský panel pro správu uživatelů a obsahu

Technické cíle

  • Postavit aplikaci na Next.js 15 s App Routerem a TypeScriptem
  • Propojit aplikaci s PostgreSQL databází přes Prisma ORM
  • Vytvořit RESTful API s více než 10 endpointy
  • Nasadit aplikaci do produkce na Vercel s automatickým CI/CD
  • Zajistit responzivní design pro mobilní i desktopová zařízení

3. Datový model (databázové schéma)

Aplikace využívá relační databázi PostgreSQL přístupnou přes Prisma ORM. Schéma databáze je definováno v souboru prisma/schema.prisma a obsahuje celkem 6 modelů.

Přehled modelů

Model Popis Klíčové atributy
User Uživatel aplikace id, name, email, role, image, emailVerified
Account OAuth účet uživatele (NextAuth) provider, providerAccountId, access_token
Session Aktivní session uživatele (NextAuth) sessionToken, userId, expires
VerificationToken Token pro ověření e-mailu / OTP identifier, token, expires

Schéma Prisma

// NextAuth modely
model User {
  id            String     @id @default(cuid())
  name          String?
  email         String?    @unique
  emailVerified DateTime?
  image         String?
  role          String     @default("user") // "admin" nebo "user"
  accounts      Account[]
  sessions      Session[]
  favorites     Favorite[]
  createdTips   Tip[]
}

model Tip {
  id          String     @id @default(cuid())
  category    String     // SLEEP, NUTRITION, MOVEMENT, STRESS
  title       String
  description String
  createdAt   DateTime   @default(now())
  updatedAt   DateTime   @updatedAt
  createdById String?
  createdBy   User?      @relation(fields: [createdById], references: [id], onDelete: SetNull)
  favorites   Favorite[]
}

model Favorite {
  id        String   @id @default(cuid())
  userId    String
  tipId     String
  createdAt DateTime @default(now())

  user User @relation(fields: [userId], references: [id], onDelete: Cascade)
  tip  Tip  @relation(fields: [tipId], references: [id], onDelete: Cascade)

  @@unique([userId, tipId])
}

Vztahy mezi modely

  • User → Account: 1:N – uživatel může mít více OAuth účtů (Google, aj.)
  • User → Session: 1:N – uživatel může mít více aktivních sessions
  • User → Tip: 1:N – uživatel může vytvořit více tipů
  • User → Favorite: 1:N – uživatel může mít více oblíbených tipů
  • Tip → Favorite: 1:N – tip může být oblíbený u více uživatelů
  • User ↔ Tip (přes Favorite): M:N – uživatelé a tipy jsou propojeni přes pivot tabulku Favorite

Kaskádové mazání je nastaveno tak, že při smazání uživatele se automaticky smažou jeho sessions, OAuth účty a oblíbené tipy. Tipy vytvořené smazaným uživatelem zůstanou zachovány (createdById se nastaví na NULL).

4. Use Case

Aplikace rozlišuje tři typy uživatelů: Nepřihlášený návštěvník, Přihlášený uživatel a Administrátor.

Nepřihlášený návštěvník

  • Prohlíží tipy na úvodní stránce
  • Filtruje tipy podle kategorií (Sleep, Nutrition, Movement, Stress)
  • Vyhledává tipy pomocí full-text vyhledávání
  • Přihlásí se pomocí Google OAuth nebo e-mailového OTP

Přihlášený uživatel

  • Vše co nepřihlášený návštěvník, plus:
  • Přidá tip do oblíbených / odebere z oblíbených
  • Zobrazí svou stránku oblíbených tipů
  • Vytvoří vlastní tip (title, description, category)
  • Smaže vlastní tip
  • Zobrazí stránku „Moje tipy"
  • Odhlásí se

Administrátor

  • Vše co přihlášený uživatel, plus:
  • Zobrazí administrátorský panel
  • Prohlíží statistiky databáze (počty uživatelů, tipů, oblíbených, tipy dle kategorií)
  • Spravuje uživatele – zobrazí seznam, změní roli (user ↔ admin), smaže uživatele
  • Smaže jakýkoli tip v systému

5.1 Frameworky

Next.js 15 (App Router)

Next.js je hlavní framework aplikace [1]. Využívá tzv. App Router, který umožňuje definovat stránky jako React Server Components a minimalizuje tak množství JavaScriptu odesílaného do prohlížeče [1]. Každá stránka je serverově renderována, což zajišťuje rychlé první načtení a dobrou SEO optimalizaci. API Routes fungují jako serverless funkce na platformě Vercel [7].

NextAuth.js

NextAuth.js zajišťuje veškerou logiku autentizace [2]. Podporuje Google OAuth provider pro přihlášení přes Google účet a vlastní e-mailový flow pro OTP autentizaci. Session management je řešen pomocí databázových sessions uložených v PostgreSQL (modely Session a Account). NextAuth.js také automaticky řeší CSRF ochranu a bezpečné cookies [2].

Prisma ORM

Prisma slouží jako vrstva mezi aplikací a PostgreSQL databází [3]. Schéma databáze je definováno v souboru prisma/schema.prisma. Prisma generuje typově bezpečný klient pro TypeScript, čímž eliminuje SQL injection útoky a zajišťuje konzistenci dat [3]. Pro seedování testovacích dat slouží skript prisma/seed.ts.

shadcn/ui + Radix UI

shadcn/ui je kolekce přístupných a stylizovatelných UI komponent postavených na primitivách knihovny Radix UI [5, 6]. Komponenty jsou kopírovány přímo do projektu (nejsou závislostí npm balíčku), což umožňuje jejich plnou customizaci pomocí Tailwind CSS. V projektu jsou využívány komponenty Button, Card, Avatar, Dialog a DropdownMenu.

Tailwind CSS

Tailwind CSS je utility-first CSS framework, který umožňuje stylizovat komponenty přímo v JSX bez nutnosti psát separátní CSS soubory [4]. Díky JIT compileru jsou do výsledného bundle zahrnuty pouze skutečně použité styly, což minimalizuje velikost CSS souboru v produkci [4].

5.2 Bezpečnost

Autentizace a autorizace

  • NextAuth.js session management – sessions jsou uloženy v databázi s expirací, nikoli pouze v cookies [2]
  • Google OAuth 2.0 – přihlašování přes důvěryhodného poskytovatele identity, hesla nejsou uložena v aplikaci [13]
  • Email OTP – jednorázové kódy s časovou platností odeslané na e-mail uživatele přes Nodemailer [8]
  • CSRF ochrana – NextAuth.js automaticky generuje a ověřuje CSRF tokeny [2]
  • Secure & HttpOnly cookies – session cookies jsou chráněny před čtením z JavaScriptu [12]
  • Role-based access control (RBAC) – administrátorské endpointy ověřují roli admin v databázi při každém požadavku [12]

Databáze

  • Prisma ORM – všechny dotazy jsou parametrizované, SQL injection je znemožněn na úrovni ORM [3, 12]
  • SSL šifrované připojení – komunikace s PostgreSQL probíhá přes SSL/TLS [7]
  • Validace vstupů – délka titulku (1–100 znaků), délka popisu (1–500 znaků) a platná kategorie jsou ověřovány na serveru před uložením do databáze [12]
  • Kaskádové mazání – při smazání uživatele se automaticky odstraní jeho sessions a oblíbené tipy [3]

API ochrana

  • Každý chráněný endpoint ověřuje platnou session pomocí getServerSession() [2]
  • Vlastnictví tipu je ověřeno před smazáním – pouze autor nebo admin může smazat tip
  • Admin nemůže smazat vlastní účet ani odebrat sobě roli admina (ochrana před lockoutem)

Prostředí a nasazení

  • Citlivé hodnoty (API klíče, databázové heslo, NEXTAUTH_SECRET) jsou uloženy jako environment variables na Vercelu, nikoli v kódu [7]
  • Soubor .env je zahrnut v .gitignore
  • Vercel poskytuje automatický HTTPS/SSL certifikát, DDoS ochranu a edge network security [7]

6. Příklady implementace

6.1 API endpoint – vyhledávání tipů (GET /api/tips)

Endpoint přijímá query parametry pro full-text vyhledávání, filtrování kategorií a stránkování. Sestavuje Prisma dotaz dynamicky podle zadaných filtrů.

// app/api/tips/route.ts
export async function GET(req: Request) {
  const session = await getServerSession(authOptions);
  const { searchParams } = new URL(req.url);

  // Načtení parametrů z URL
  const search = searchParams.get("search") || "";
  const categories = searchParams.get("categories")?.split(",").filter(Boolean) || [];
  const searchIn = searchParams.get("searchIn") || "both"; // "title" | "description" | "both"
  const page = parseInt(searchParams.get("page") || "1");
  const limit = parseInt(searchParams.get("limit") || "9");

  // Dynamické sestavení WHERE podmínky
  const whereConditions: any[] = [];

  // Filtr kategorií
  if (categories.length > 0) {
    whereConditions.push({ category: { in: categories } });
  }

  // Full-text vyhledávání
  if (search.trim()) {
    const searchConditions: any[] = [];
    if (searchIn === "title" || searchIn === "both") {
      searchConditions.push({ title: { contains: search } });
    }
    if (searchIn === "description" || searchIn === "both") {
      searchConditions.push({ description: { contains: search } });
    }
    if (searchConditions.length > 0) {
      whereConditions.push({ OR: searchConditions });
    }
  }

  const where = whereConditions.length > 0 ? { AND: whereConditions } : undefined;

  // Dotaz do databáze s stránkováním
  const total = await prisma.tip.count({ where });
  const tips = await prisma.tip.findMany({
    where,
    orderBy: { createdAt: "desc" },
    skip: (page - 1) * limit,
    take: limit,
  });

  return NextResponse.json({ tips, total, page, totalPages: Math.ceil(total / limit) });
}

6.2 API endpoint – toggle oblíbených (POST /api/favorites)

Endpoint funguje jako toggle – pokud tip již je oblíbený, odebere ho; pokud není, přidá ho. Využívá Prisma @@unique constraint pro detekci duplicit.

// app/api/favorites/route.ts
export async function POST(req: Request) {
  const session = await getServerSession(authOptions);

  // Ověření přihlášení
  if (!session?.user?.id) {
    return NextResponse.json({ error: "Unauthorized" }, { status: 401 });
  }

  const { tipId } = await req.json();

  // Zjistíme, zda tip již je v oblíbených
  const existing = await prisma.favorite.findUnique({
    where: { userId_tipId: { userId: session.user.id, tipId } },
  });

  if (existing) {
    // Tip je oblíbený → odebereme ho
    await prisma.favorite.delete({ where: { id: existing.id } });
    return NextResponse.json({ favorited: false });
  } else {
    // Tip není oblíbený → přidáme ho
    await prisma.favorite.create({
      data: { userId: session.user.id, tipId },
    });
    return NextResponse.json({ favorited: true });
  }
}

6.3 Email OTP autentizace (POST /api/auth/send-otp)

Endpoint vygeneruje náhodný šestimístný OTP kód, uloží ho do databáze a odešle uživateli e-mailem přes Nodemailer.

// app/api/auth/send-otp/route.ts (zjednodušeno)
export async function POST(req: Request) {
  const { email } = await req.json();

  // Generování OTP kódu
  const otp = Math.floor(100000 + Math.random() * 900000).toString();
  const expires = new Date(Date.now() + 10 * 60 * 1000); // platnost 10 minut

  // Uložení OTP tokenu do databáze
  await prisma.verificationToken.upsert({
    where: { identifier_token: { identifier: email, token: otp } },
    create: { identifier: email, token: otp, expires },
    update: { token: otp, expires },
  });

  // Odeslání e-mailu přes Nodemailer
  await transporter.sendMail({
    from: process.env.EMAIL_FROM,
    to: email,
    subject: "Váš přihlašovací kód",
    html: `<p>Váš jednorázový kód je: <strong>${otp}</strong></p>
           <p>Kód platí 10 minut.</p>`,
  });

  return NextResponse.json({ success: true });
}

10. Závěr

Úspěšnost splnění cílů

Projekt Healthy Lifestyle Tips byl úspěšně dokončen a splňuje všechny zadané požadavky. Aplikace je plně funkční, nasazená v produkci na platformě Vercel a dostupná na adrese https://healthy-lifestyle-tips.vercel.app/.

Co se naučil tým

Projekt přinesl cenné praktické zkušenosti s vývojem moderní full-stack aplikace. Tým se seznámil s Next.js App Routerem, pochopil rozdíl mezi Server a Client Components, naučil se pracovat s Prisma ORM a implementovat bezpečnou autentizaci pomocí NextAuth.js.

Další příležitosti a možná rozšíření

  • Komentáře k tipům – možnost diskuze pod každým tipem
  • Hodnocení tipů – like/dislike systém nebo hvězdičkové hodnocení
  • Notifikace – e-mailové upozornění na nové tipy v oblíbených kategoriích
  • Automatizované testy – implementace unit a integration testů (Jest, Playwright)
  • Pokročilé vyhledávání – full-text search přímo v PostgreSQL pomocí pg_trgm
  • Vícejazyčnost – lokalizace aplikace do angličtiny a dalších jazyků pomocí next-intl
  • PWA – přeměna aplikace na Progressive Web App pro offline podporu
